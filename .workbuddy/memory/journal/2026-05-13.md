# 2026-05-13

## 修复 Python 项目启动失败

**问题**: `terminal_chat.py` 和 `agent_memory_system.py` 无法启动，报 `ImportError: cannot import name 'SyncClient' from 'openviking'`

**根因**:
1. 代码使用 `openviking` 的 `SyncClient` / `VikingClient` API，但安装的 v0.3.16 版本已更名为 `SyncOpenViking` / `OpenViking`
2. 即使改用新 API，`SyncOpenViking.initialize()` 在 Windows 上因缺少原生后端 (`PersistStore`) 而崩溃
3. `openviking` 在 Windows 平台本地模式完全不可用

**解决方案**: 将 openviking 依赖替换为纯本地 JSON 文件存储方案：
- `terminal_chat.py`: 新增 `LocalMemory` 类，用 `memory_store/` 目录存储对话历史和知识库，关键词搜索替代向量检索
- `agent_memory_system.py`: 同样改为本地文件存储，`agent_memory_store/` 目录
- `requirements.txt`: 添加 `python-dotenv>=1.0.0`，不再依赖 openviking

**保留的功能**: DeepSeek API 对话、思考过程展示、长期记忆存储/检索、RAG 文档知识库（改为关键词匹配）

## 彻底移除 OpenViking

完全卸载 openviking 包 (v0.3.16) 并清理配置 (`~/.openviking/`) 和工作区数据 (`OpenViking-Workspace-New/`)。Windows 上 Native Engine 始终不可用，HTTP 服务端模式同样依赖 PersistStore 初始化失败。

## 搭建渐进式记忆架构

采用分层 MD 文件方案替代 OpenViking：
- `INDEX.md` (L0): 全局索引，每次会话读取
- `agents/` (L1): 每个 Agent 的摘要（game-dev, accounting, tools）
- `topics/` (L2): 专题深入文档
- `journal/` (L3): 每日流水，30天轮转
- `MEMORY.md`: 长期事实（就地更新）

## 终端聊天升级渐进式记忆

将 `terminal_chat.py` 的 `LocalMemory` (扁平 JSON) 替换为 `ProgressiveMemory` (分层 MD)：
- `progressive_memory.py` 新建，包含完整的渐进检索、分层读写、索引重建
- `terminal_chat.py` 重构：新增 memory/topic/index 命令，系统提示注入 INDEX 概览
- Agent 每次启动加载 INDEX.md (L0) 到 system prompt，用户提问时自动从 agents/topics 展开相关记忆
- 对话记录写入 journal/ (L3)，RAG 知识库保留不变
